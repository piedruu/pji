Rede de Computadores

O que é uma rede de computadores?

Uma rede de computadores é um conjunto de dispositivos (computadores, celulares, impressoras etc.) conectados entre si para compartilhar informações e recursos, como arquivos, internet e impressoras.

Exemplo avançado

Em uma infraestrutura corporativa de alta disponibilidade, redes SD-WAN (SoftwareDefined Wide Area Network) são implementadas para otimizar o tráfego entre múltiplos data centers e filiais. A comunicação entre segmentos da rede é gerenciada por políticas de QoS (Quality of Service) que priorizam aplicações críticas. O roteamento dinâmico BGP (Border Gateway Protocol) é usado para redundância, enquanto o MPLS (Multiprotocol Label Switching) garante baixa latência em conexões privadas.

Exemplo intermediário

Uma empresa utiliza VLANs (Virtual Local Area Networks) para segmentar sua rede interna, separando tráfego de departamentos distintos, como RH e TI. O firewall configurado com regras de ACL (Access Control List) restringe acessos entre segmentos. Para otimizar a segurança, a autenticação 802.1X garante que apenas dispositivos autorizados acessem a rede corporativa.

Exemplo básico

Em casa, quando você acessa a internet no seu celular, ele se conecta ao roteador WiFi, que recebe e envia dados da internet para o seu dispositivo. O roteador funciona como um “correio”, enviando informações para o lugar certo, garantindo que seu celular consiga abrir vídeos, redes sociais e sites.

Evolução Histórica

Anos 1960 - O Início: ARPANET

O Departamento de Defesa dos EUA criou a ARPANET para conectar centros de pesquisa. Foi a primeira rede a usar comutação de pacotes, que divide dados em partes menores para envio eficiente. Essa tecnologia foi a base do que se tornaria a Internet.

Anos 1970 - Criação de Protocolos

Surgiram os primeiros protocolos de rede, especialmente o TCP/IP, criado em 1974. Esses protocolos permitem que diferentes sistemas se comuniquem. O e-mail também surgiu nessa época, revolucionando a comunicação digital.

Anos 1980 - Redes Locais (LAN) e Expansão

Com a popularização das redes locais (LANs), empresas e escolas começaram a conectar seus computadores internamente. O padrão Ethernet e o uso de cabos coaxiais marcaram essa fase, tornando as redes mais acessíveis fora do meio acadêmico.

Anos 1990 - A Internet Comercial e a Web

A criação da World Wide Web (WWW) e dos navegadores, como o Netscape, transformou a Internet em algo visual e acessível. A Internet se expandiu para o público geral, com acesso por linha telefônica e depois por banda larga.

Anos 2000 - Redes sem Fio e Mobilidade

Com a chegada do Wi-Fi, notebooks e celulares começaram a se conectar sem fios. A internet móvel, com redes 3G e 4G, trouxe acesso em qualquer lugar. As redes sociais e os serviços de streaming também se tornaram populares.

Anos 2010 até Hoje - Conectividade Total

O uso de tecnologias como 5G, fibra ótica, computação em nuvem e Internet das Coisas (IoT) conectou praticamente tudo. Hoje, a internet é essencial para trabalho, estudo, entretenimento e vida social.


As redes de computadores evoluíram de simples conexões militares para uma infraestrutura global que conecta bilhões de dispositivos. A inovação continua, impulsionando novas formas de viver e trabalhar.

Tipos de redes

Classificação por abrangência

LAN (Local Area Network): Redes locais usadas em residências, escritórios ou pequenos ambientes

MAN (Metropolitan Area Network): Abrange áreas maiores, como cidades ou campus universitários.

WAN (Wide Area Network): Redes de grande escala, cobrindo cidades, países ou até o mundo.

Exemplo avançado

Empresas globais utilizam redes WAN híbridas que combinam conexões privadas (como MPLS) com VPNs IPsec para garantir segurança e eficiência. A tecnologia SDN (Software-Defined Networking) oferece controle centralizado, permitindo balanceamento de carga e failover em caso de falhas.

Exemplo intermediário

Universidades com redes MAN conectam prédios usando fibra óptica, otimizando o tráfego por switches avançados.

Exemplo básico

O Wi-Fi de uma escola é uma LAN; se várias escolas na mesma cidade estiverem conectadas, formam uma MAN, e a internet é um exemplo de WAN.

Classificação por topologia

A topologia de uma rede define a forma como os dispositivos estão conectados. Aqui estão as principais:

  • Estrela - Todos os dispositivos se conectam a um ponto central (como um roteador Wi-Fi).
  • Anel - Os dispositivos se conectam em um círculo fechado.
  • Barramento - Todos os dispositivos podem se conectar diretamente uns aos outros.
  • Malha - Todos os dispositivos podem se conectar diretamente uns aos outros.

Exemplo avançado

Grandes data centers utilizam a topologia malha híbrida, onde múltiplos links redundantes garantem alta disponibilidade. Essa configuração permite failover automático e otimização de tráfego via protocolos como OSPF (Open Shortest Path First) e STP (Spanning Tree Protocol).

Exemplo intermediário

Um provedor de internet pode usar uma topologia em malha para garantir conexões redundantes entre servidores, evitando pontos únicos de falha. Em uma fábrica, uma topologia barramento pode conectar sensores e máquinas na linha de produção.

Exemplo básico

Se você tem um roteador Wi-Fi em casa e todos os aparelhos se conectam a ele, isso é uma topologia estrela. Se os computadores estiverem ligados uns aos outros sem precisar de um roteador central, isso é topologia anel.

Principais protocolos de rede

  • IP (Internet Protocol) - Endereça os dispositivos na rede (como números de telefone para os computadores).
  • TCP (Transmission Control Protocol) - Garante que os dados cheguem corretamente.
  • UDP (User Datagram Protocol) - Mais rápido que o TCP, mas sem garantia de entrega (usado em chamadas de vídeo)
  • HTTP/HTTPS - Usado para acessar sites na internet.
  • FTP (File Transfer Protocol) - Usado para transferência de arquivos.

    • Exemplo avançado

    Uma aplicação distribuída baseada em microserviços pode utilizar gRPC sobre HTTP/2 para comunicação eficiente entre serviços. Além disso, pode empregar MQTT para dispositivos IoT e QUIC para otimizar conexões web de baixa latência.

    Exemplo intermediário

    Servidores web utilizam HTTP/HTTPS para entregar páginas da web, enquanto o protocolo DNS converte nomes de domínio (ex: google.com) em endereços IP. Aplicações em tempo real, como chamadas VoIP, usam UDP para priorizar velocidade sobre confiabilidade

    Exemplo básico

    Quando você acessa um site, o navegador usa HTTP para buscar a página. Se for um site seguro, será HTTPS. Quando você assiste a um vídeo no YouTube, ele usa UDP para carregar mais rápido sem travar.
